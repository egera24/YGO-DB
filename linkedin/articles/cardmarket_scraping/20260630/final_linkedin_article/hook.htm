I spent weeks scraping Cardmarket for a side project. Then I found out I didn't have to.

HTTP 429. IP bans. Chrome automation running for hours on my laptop. Cloudflare winning every round.

Then, buried in Cardmarket's infrastructure: bulk JSON files on S3. No bot mitigation. No HTML parsing. Full catalog, price guide included. Download time: minutes.

I wrote up the full story — the 4-step legacy scraper, why Cloudflare made it unsustainable, the S3 pipeline that replaced it, and the catalog matching logic that turned out to be the harder problem.

Here's what the journey looked like:
💥 Legacy pipeline: browser automation via CDP, ~0.05 req/s, datacenter IPs blocked before the first request
💥 New pipeline: weekly S3 download → Yugipedia catalog join → automated import via GitHub Actions
💥 The honest trade-off: ~40 manual set aliases, price-ordered rarity inference, per-set rejection logs

One problem traded for another — but at least this one has structure.

Full article linked below. If you've ever joined two catalogs with no shared IDs and different naming conventions, I'd like to know how you handled it. ↓

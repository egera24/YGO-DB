Because the rarity is not part of the attributes, we have to best guess the rarity of the card based on the prices and parallel we should assign the prices to the cards in the database as well.

In the database we have the details of all cards, the rarity as well. 

The logic for finding the correct expansion name and card name is the following.

1) For every yugipedia tcg set a cardmarket expansion id should be found and based on that the cardmarket_expansions table should be filled.

Take as an example the Rarity Collection 5 in the database. It should check, which blocks contain the the "name" Rarity Collection 5 in the products_nonsingles_3.json file.
This search should be as a like operator in SQL, so the system will find 4 blocks and all of the blocks should contain the idExpansion 6424. 
Here there should be a validation if every result contains the same idExpansion. If not, an error should be raised and the user should be informed, the process should be terminated.

Now we know which cardmarket idExpansion which yugioh expansion code belongs to.

2) In the database we have all of the yugipedia card printings. Now we know which cardmarked IdExpansion belongs to which yugipedia expension. 
The file products_singles_3.json contains all of the cards from cardmarket. We can identify the cards by the idExpension and by the card name. 
However the rarity is still missing which is a needed identification for a card printing. 
Based on cardmarket catalogs the only available option is to best guess the rarity. 
The price_guide_3.json file contains the prices of the cards. We should best guess the rarity of the card and parallel we can store the prices as well.

That way if an expension has more than one printig of the given card, based on the available rarities we should be able to assign the prices to the correct card printing.
If during the process there is a mismatch, an error should be raised and the user should be informed, the process should be terminated.
What can be a mismatch: there are more card printings in yugipedia catalog than printings/prices in the cardmarket catalog or vice versa.
